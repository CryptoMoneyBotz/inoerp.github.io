Work Order

Edit Document

Work orders are always created with reference to a work definition. A work definition combines the material requirement, operations and resource requirement. When work orders are created with reference to a work definition, system copies all the data from the work definition. System creates all the work orders in draft status. You can review and modify the work order/routing/bom details of the work order before releasing it.

View/Update

To view/update an work order, navigate to the "Work Order" search screen from your dashboard/favorite.

Enter the item/organization/work order details or any other criteria in the search form and click on the search button.

If you don't enter item/organization details, the system will show you all the work orders. If your organization has configured to make any field mandatory for search then you have to enter those fields to search work orders

The application will fetch all the data from the server and will take you to the search result scree. The search result screen looks a bit different in desktop and mobile devices.

Search result on a mobile device:

To view the search result in a table format, click on the table icon available on left bottom corner.

To open a specific work order, click on the menu of that record, and click on view / fetch & view.

System will fetch all the data related to the selected work order. Depending on the device size, the work order details are either shown right to the search result screen or in a separate screen.

Work order on a mobile device:

You can also click on the expand icon against any order on the search result screen to view the work order header details such as work order number, quantities, start date, status, etc.

You can modify any field that is available for update and click on the save button to save the changes. The app will post all the data to the server and will show the messages returned from the server.

Scroll left, right, up, and down view all the tabs and fields.

The default configuration shows all the operations, material requirement and resource requirements in separate tabs. You can also configure the application to show other tabs with details such as

  • completed operations against the work order
  • issued materials against the work order
  • issued resources against the work order

Work Line tab shows all the operations available for thw work order. You can add extra operation or modify the operations copied from the work definition. You can view material requirement for a specific operation by clicking on the material button available on Links heading. You can also view all material requirement for the entire work order by navigating to the material requirement tab available next to the line tab.

You can view resource requirement for a specific operation by clicking on the resource button available on Links heading. You can also view all resource requirement for the entire work order by navigating to the resource requirement tab available next to the line tab.

Navigate to material tab to view all the material requirement for the job. By default the application shows the details in an expandable list view. However, if you are working on a large screen device, click on the table icon visible on right bottom to view all the details in a table format. On this tab, you can modify material requirement details.

Navigate to resource tab to view all the resource requirement for the job. By default the application shows the details in an expandable list view. However, if you are working on a large screen device, click on the table icon visible on right bottom to view all the details in a table format. On this tab, you can modify resource requirement details.

Click on the back button to navigate back to the search result screen.

Copy

To copy an work order,first search the work order, and then click on the copy button on the search result. Copy option is also available on work order action list of values.

System copies all the information from the selected work order except those fields marked as not copied. You can configure which fields should not be copied as per your business requirements. If change the work definition before saving the work order the application will copy all operation and operation details of the selected work definition.

By default, the app is configured not to copy start date, quantities, etc. However, you can change all these settings as per your business requirements. Enter all the non-copied mandatory fields such as date, quantity.

Calendar icons are available next to all date / date-time fields. Click on the calendar icon to select a date.

Click on the time icon to select/change time.

Enter/update any other fields on work order header, and click on the save button. System will post all the details to the server and will show you below message if the work order is successfully created

Navigate back to the search screen to search the new work order.

Review the work order details. Scroll left, right, up, and down view all the tabs and fields, and make modifications as required before confirming the work order

Click on the back button to navigate back to the search result screen.

Create

To copy an work order,first navigate to the work order search screen, and then click on the add new button on the search screen. Add new option is also available on work order action list of values.

Select the work definition, and other details. The primary difference between Create and Copy is that Create shows a blank form whereas copy form copies values from the selected work order. So, copy option is always preferable compared to create new

Click on the select icon next to the work definition, system will show a work definition search form in a popup. Enter work definition details and click on the search button. Click on the select icon on the first column to select any work definition.

System defaults all the values available on work definition including the work definition name, item number, organization code, etc.

If you click on the save button without entering all the mandatory fields then system will show below error message.

Enter all the other details, and click on the save button to create a new work order.

Confirm

To confirm an work order, open the work order and navigate to action tab. The action tab shows below buttons by default

  • Confirm
  • Cancel
  • Close
  • Pending Close
  • Hold You can also define other actions and corresponding business logics as per your requirement.

Click on the confirm button to confirm a work order. If the work order is confirmed then system will show below message.

Re-open the work order and check the work order status.

By default, the work application makes many fields readonly after confirmation. However, none of these creates any technical constraints on the application process. So, you can change the logic as per your business requirement by changing below javascript function.
